---
title: "Snapshot: createSnapshot and restoreSnapshot"
description: "Reference for createSnapshot and restoreSnapshot — the tabletop-engine API for saving and restoring versioned deep clones of canonical game state."
---

Snapshots let you capture the full canonical state of a running game into a plain serializable object, store it wherever you like, and restore it later with a guaranteed deep clone. The snapshot format includes a version field so the engine can handle migrations in future releases. Both functions operate on `CanonicalState` — the `{ game, runtime }` object the executor produces — and are transport-agnostic: you decide how to persist or transfer the JSON.

## `createSnapshot(state)`

```typescript
function createSnapshot<State extends CanonicalState>(
  state: State,
): Snapshot<State>
```

Creates a deep clone of `state` and wraps it in a versioned `Snapshot` envelope. The original state object is not modified.

<ParamField path="state" type="CanonicalState" required>
  The canonical state object returned by the executor. Typically
  `result.state` after a command execution, or the value from
  `createInitialState()`.
</ParamField>

<ResponseField name="Snapshot<State>" type="object">
  <Expandable title="fields">
    <ResponseField name="version" type="1">
      Always `1` in the current release.
    </ResponseField>
    <ResponseField name="state" type="State">
      A deep clone of the state passed to `createSnapshot`. Mutations to the
      original state do not affect the snapshot, and mutations to
      `snapshot.state` do not affect the original.
    </ResponseField>
  </Expandable>
</ResponseField>

## `restoreSnapshot(snapshot)`

```typescript
function restoreSnapshot<State extends CanonicalState>(
  snapshot: Snapshot<State>,
): State
```

Recovers state from a snapshot by returning a deep clone of `snapshot.state`. The snapshot object is not modified.

<ParamField path="snapshot" type="Snapshot<State>" required>
  A snapshot previously created by `createSnapshot`, or deserialized from
  JSON storage. The `version` field must be `1`.
</ParamField>

## `Snapshot<State>`

```typescript
interface Snapshot<State extends CanonicalState = CanonicalState> {
  version: 1;
  state: State;
}
```

A plain, JSON-serializable envelope. Store `snapshot.state` as a JSON column in a database; reconstruct the full snapshot object by wrapping it with `{ version: 1, state: <stored value> }` before calling `restoreSnapshot`.

## Usage

```typescript
import { createSnapshot, restoreSnapshot } from "tabletop-engine";

// After executing a command, capture a checkpoint
const result = executor.executeCommand(state, command);
const snapshot = createSnapshot(result.state);

// Serialize and store
await db.saveSnapshot(gameId, JSON.stringify(snapshot.state));

// Later: reload and restore
const raw = await db.loadSnapshot(gameId);
const restored = restoreSnapshot({ version: 1, state: JSON.parse(raw) });

// Continue the game from the restored state
const nextResult = executor.executeCommand(restored, nextCommand);
```

## Use cases

- **Persistence between server restarts** — snapshot state after each command and reload it on startup.
- **Checkpointing** — store a snapshot at the start of each round so you can roll back to a known-good state during debugging.
- **Replay construction** — snapshots serve as the `initialSnapshot` field in `ReplayRecord`. See [Replay helpers](/reference/testing-apis/replay-helpers) for details.
- **Testing** — snapshot a complex hand-crafted state to reuse it across multiple test cases without re-running setup.

<Tip>
  Both `createSnapshot` and `restoreSnapshot` use `structuredClone` internally,
  so the cloned state is fully independent of the original. You can safely
  mutate the restored state without affecting the snapshot you stored.
</Tip>
