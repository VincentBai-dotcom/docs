---
title: "GameDefinitionBuilder API reference"
description: "Build a complete GameDefinition by chaining .rootState(), .initialStage(), .setupInput(), and .setup() on GameDefinitionBuilder, then call .build()."
---

`GameDefinitionBuilder` is the entry point for assembling a game. You call it with a game name, attach a root state class, register an initial stage, optionally define setup input and a setup callback, then call `.build()` to produce a sealed `GameDefinition` that the executor and CLI can consume.

## Constructor

```typescript
new GameDefinitionBuilder(name: string)
```

Creates a new builder for a game with the given name.

<ParamField path="name" type="string" required>
  A unique identifier for the game. This value is embedded in the produced `GameDefinition` and used as a reference by the CLI and protocol generation tools.
</ParamField>

## Methods

### `.rootState(stateClass)`

Registers the root `@State()` class. The builder infers `FacadeGameState` and `CanonicalGameState` from this class, so you must call this before `.build()`.

<ParamField path="stateClass" type="StateClass<NextFacadeGameState>" required>
  A class decorated with `@State()`. All fields decorated with `@field(...)` become part of the canonical and facade game state.
</ParamField>

<ResponseField name="returns" type="GameDefinitionBuilder<NextFacadeGameState, ...>">
  A re-typed builder instance with `FacadeGameState` and `CanonicalGameState` set to the types derived from the provided class. All subsequent chained calls operate on the new types.
</ResponseField>

```typescript
import { GameDefinitionBuilder } from "tabletop-engine";
import { MyGameState } from "./state";

const builder = new GameDefinitionBuilder("my-game").rootState(MyGameState);
```

### `.setupInput(schema)`

Defines a schema for the input object passed to `createInitialState` at game creation time. Call this when your game requires configuration (such as player count or house rules) at the point the game is instantiated.

If you do not call `.setupInput()`, `createInitialState` accepts only an RNG seed.

<ParamField path="schema" type="ObjectFieldType<Record<string, FieldType>>" required>
  An object schema built with `t.object(...)`. Must be of kind `"object"`; passing any other schema kind throws `"setup_input_schema_must_be_object"`.
</ParamField>

<ResponseField name="returns" type="GameDefinitionBuilder<..., SetupInputFromSchema<TSchema>>">
  A re-typed builder with the `SetupInput` type parameter inferred from the provided schema.
</ResponseField>

```typescript
import { t } from "tabletop-engine";

const builder = new GameDefinitionBuilder("my-game")
  .rootState(MyGameState)
  .setupInput(
    t.object({
      playerCount: t.number(),
    }),
  );
```

### `.initialStage(stage)`

Sets the starting stage for the game. The builder traverses the stage graph from this stage to discover all reachable stages and commands when `.build()` is called.

<ParamField path="stage" type="StageDefinition" required>
  A stage definition produced by a stage builder's `.build()` method. All stages reachable from this stage via their `.nextStages()` resolvers are registered automatically.
</ParamField>

<ResponseField name="returns" type="this">
  The same builder instance for chaining.
</ResponseField>

```typescript
const builder = new GameDefinitionBuilder("my-game")
  .rootState(MyGameState)
  .initialStage(playerTurnStage);
```

### `.setup(fn)`

Registers an optional setup callback that runs once immediately after the initial canonical state is created, before the stage machine initializes. Use this to populate starting values that depend on the RNG or setup input.

<ParamField path="fn" type="(context: GameSetupContext<FacadeGameState, SetupInput>) => void" required>
  A function called with a `GameSetupContext` containing:

  - `context.game` — a mutable facade view of the game state
  - `context.runtime` — the initial runtime state
  - `context.rng` — the deterministic RNG, seeded from the value passed to `createInitialState`
  - `context.input` — the typed setup input, or `undefined` if `.setupInput()` was not called
</ParamField>

<ResponseField name="returns" type="this">
  The same builder instance for chaining.
</ResponseField>

```typescript
const builder = new GameDefinitionBuilder("my-game")
  .rootState(MyGameState)
  .initialStage(playerTurnStage)
  .setup(({ game, rng, input }) => {
    game.playerCount = input.playerCount;
    game.firstPlayerId = rng.pick(game.playerIds);
  });
```

### `.build()`

Validates the builder configuration and produces a sealed `GameDefinition`. Throws if `.rootState()` or `.initialStage()` have not been called.

The builder performs these steps internally:

1. Collects all reachable stages via depth-first traversal from the initial stage.
2. Compiles the command map from all `activePlayer` and `multiActivePlayer` stages.
3. Compiles the canonical and runtime state schemas from the root state class.
4. Validates the default canonical state against the compiled schema.

<ResponseField name="returns" type="GameDefinition<CanonicalGameState, FacadeGameState, Commands, SetupInput>">
  A sealed game definition object. Pass this directly to `createGameExecutor` or to `defineConfig`.
</ResponseField>

**Throws:**

- `"root_state_required"` — `.rootState()` was never called.
- `"initial_stage_required"` — `.initialStage()` was never called.
- `"duplicate_stage_id:<id>"` — Two different stage objects share the same `id`.
- `"duplicate_command_id:<id>"` — Two different command objects share the same `commandId`.

```typescript
const game = new GameDefinitionBuilder("my-game")
  .rootState(MyGameState)
  .initialStage(playerTurnStage)
  .build();
```

## Full example

```typescript
import { GameDefinitionBuilder, t } from "tabletop-engine";
import { MyGameState } from "./state";
import { playerTurnStage } from "./stages";

const game = new GameDefinitionBuilder("my-game")
  .rootState(MyGameState)
  .setupInput(
    t.object({
      playerCount: t.number(),
    }),
  )
  .initialStage(playerTurnStage)
  .setup(({ game, rng, input }) => {
    game.playerCount = input.playerCount;
    game.currentPlayerId = rng.pick(game.playerIds);
  })
  .build();
```

## `defineConfig`

`defineConfig` is a helper imported from `tabletop-engine/config`. It wraps a built `GameDefinition` with CLI output options and is read by the `tabletop-engine` CLI during code generation.

```typescript
import { defineConfig } from "tabletop-engine/config";
import { game } from "./game";

export default defineConfig({
  game,
  outDir: "./generated",
});
```

<ParamField path="config.game" type="GameDefinition" required>
  The built game definition produced by `GameDefinitionBuilder.build()`.
</ParamField>

<ParamField path="config.outDir" type="string">
  The directory where the CLI writes generated protocol descriptors and client types. The path is resolved relative to the config file. When omitted, the CLI defaults to a `generated/` directory next to the config file.
</ParamField>
