tabletop-engine is a TypeScript runtime library that gives you the building blocks for a complete board game or tabletop rules engine. You define your game state with decorators, model turn structure with stages, author commands with validation and execution logic, and generate WebSocket protocol descriptors — all without committing to a specific transport, framework, or hosting platform.

## Key capabilities

tabletop-engine covers the runtime mechanics that every board game backend needs, so you can stay focused on your game's rules.

<CardGroup cols={2}>
  <Card title="State facades" icon="layer-group">
    Author game state as decorated TypeScript classes using `@State()` and `@field()`. The engine compiles these into a serializable canonical schema that persists between turns.
  </Card>
  <Card title="Stages and progression" icon="arrow-right-arrow-left">
    Model your game's turn structure as a graph of named stages. Each stage controls who can act, which commands are available, and where the game goes next.
  </Card>
  <Card title="Commands" icon="terminal">
    Define commands with `validate` and `execute` hooks. The executor runs validation before applying any mutations, so invalid commands never corrupt game state.
  </Card>
  <Card title="Hidden information" icon="eye-slash">
    Project viewer-specific state with `getView()`. Each player sees only what they're allowed to see — cards in hand, private tokens, or role assignments — while the engine keeps the authoritative canonical state separate.
  </Card>
  <Card title="Deterministic RNG" icon="shuffle">
    Seeded random number generation is built into setup and execution contexts. Games are fully replayable and testable from a given seed.
  </Card>
  <Card title="Protocol generation" icon="plug">
    Generate WebSocket protocol descriptors and AsyncAPI specs directly from your game definition. Connect your engine to a hosted server without hand-writing message schemas.
  </Card>
</CardGroup>

## Who it's for

tabletop-engine is designed for TypeScript developers building the backend logic for turn-based or board game experiences. It works well whether you're:

- Building a standalone game server for a hobby project
- Creating a reusable rules engine to power multiple clients (web, terminal, mobile)
- Prototyping game mechanics before layering on a UI or multiplayer infrastructure

The library deliberately stays transport-agnostic and has no opinions about auth, lobbies, persistence, or hosting. You bring those layers; tabletop-engine handles the game rules.

<Note>
  tabletop-engine requires TypeScript 5+ and uses the `experimentalDecorators` compiler option for state facade authoring. See [Installation](/learn/quick-start/installation) for the required `tsconfig.json` settings.
</Note>
