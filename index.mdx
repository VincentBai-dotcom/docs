tabletop-engine gives you everything you need to build a complete board game or tabletop rules engine in TypeScript. Define your game state with decorators, model turn progression with stages, author commands with validation and execution logic, and generate WebSocket protocol descriptors and AsyncAPI specs — all from a single, cohesive library.

## What you can build

tabletop-engine handles the runtime mechanics so you can focus on game logic:

- **Turn-based games** with single active players (chess, Splendor, Ticket to Ride)
- **Simultaneous action games** where multiple players act at the same time
- **Hidden information games** where each player sees a different view of state
- **Hosted game servers** with generated WebSocket protocols and AsyncAPI specs

## How it works

<Steps>
  <Step title="Define your game state">
    Use `@State()` and `@field()` decorators to describe your game's data model. The engine compiles this into a type-safe canonical schema.
  </Step>
  <Step title="Model stages and commands">
    Build a progression graph using `createStageFactory()`. Each stage defines which commands are available and who can act.
  </Step>
  <Step title="Create the executor">
    Call `createGameExecutor(game)` to get a runtime that handles command validation, execution, state transitions, and event emission.
  </Step>
  <Step title="Generate protocol artifacts">
    Use the CLI (`tabletop-cli generate`) or API to produce protocol descriptors, TypeScript client SDKs, and AsyncAPI specs for your game server.
  </Step>
</Steps>
