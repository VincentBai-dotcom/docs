title: "Get started with tabletop-engine"
description: "Learn how to define a minimal game, execute a command, and read state back using tabletop-engine's GameDefinitionBuilder and createGameExecutor in minutes."
---

This guide walks you through building a minimal working game with tabletop-engine — a single-state counter that a player can increment. By the end, you'll have a complete game definition, an executor, and a working command execution loop that you can extend into a real game.

<Steps>
  <Step title="Install the package">
    Add `tabletop-engine` to your project. See [Installation](/installation) for the full `tsconfig.json` setup required for decorators.

    ```bash
    npm install tabletop-engine
    ```
  </Step>

  <Step title="Define your game state">
    Create a state facade class using the `@State()` and `@field()` decorators. The `t` object provides the schema types that the engine uses to serialize and validate your state.

    ```typescript
    import { State, field, t } from "tabletop-engine";

    @State()
    class CounterState {
      @field(t.number())
      value!: number;

      increment() {
        this.value += 1;
      }
    }
    ```

    The decorated class is an authoring-time facade — it gives you typed methods to mutate state during command execution. The engine persists the underlying plain data, not the class instance.
  </Step>

  <Step title="Define a stage and a command">
    Use `createStageFactory` to create a stage builder, then wire up a command with `validate` and `execute` hooks. A stage controls who can act and which commands are available at that point in the game.

    ```typescript
    import {
      State,
      field,
      t,
      createStageFactory,
    } from "tabletop-engine";

    @State()
    class CounterState {
      @field(t.number())
      value = 0;

      increment() {
        this.value += 1;
      }
    }

    const defineStage = createStageFactory<CounterState>();

    const incrementStage = defineStage("counting")
      .singleActivePlayer()
      .activePlayer(() => "player1")
      .commands([
        {
          commandId: "increment",
          validate: () => ({ ok: true }),
          execute: ({ game }) => {
            (game as CounterState).increment();
          },
        },
      ])
      .transition(() => incrementStage)
      .build();
    ```

    <Tip>
      The `transition` callback returns the next stage after a command executes. Returning the same stage keeps the game looping on the same player — useful for testing. In a real game you'd transition to a different stage or the next player's turn.
    </Tip>
  </Step>

  <Step title="Build the game definition">
    Assemble the state and initial stage into a `GameDefinitionBuilder`. The field initializer (`value = 0`) on `CounterState` provides the default canonical value automatically.

    ```typescript
    import { GameDefinitionBuilder } from "tabletop-engine";

    const game = new GameDefinitionBuilder("counter")
      .rootState(CounterState)
      .initialStage(incrementStage)
      .build();
    ```
  </Step>

  <Step title="Create an executor and run a command">
    Pass your game definition to `createGameExecutor` to get a runtime. Use `createInitialState` to generate the starting state, then `executeCommand` to apply a command.

    ```typescript
    import { createGameExecutor } from "tabletop-engine";

    const executor = createGameExecutor(game);

    // Create a fresh game state from a seed
    const state = executor.createInitialState("seed-123");

    // Execute the increment command as player1
    const result = executor.executeCommand(state, {
      type: "increment",
      actorId: "player1",
      input: {},
    });

    if (result.ok) {
      console.log(result.state.game.value); // 1
    }
    ```

    The executor returns an `ExecutionResult` with an `ok` flag. On success, `result.state` contains the new canonical state. On failure, `result` contains a `reason` describing what went wrong.
  </Step>

  <Step title="Read the updated state">
    The state the executor returns is plain serializable data — no class instances. You can inspect it directly, persist it, or pass it back into `executeCommand` for the next turn.

    ```typescript
    if (result.ok) {
      const newState = result.state;

      // Plain canonical game data
      console.log(newState.game.value); // 1

      // Execute another command against the new state
      const result2 = executor.executeCommand(newState, {
        type: "increment",
        actorId: "player1",
        input: {},
      });

      if (result2.ok) {
        console.log(result2.state.game.value); // 2
      }
    }
    ```
  </Step>
</Steps>

## Complete example

Here is the full counter game in a single file:

```typescript
import {
  State,
  field,
  t,
  createStageFactory,
  createGameExecutor,
  GameDefinitionBuilder,
} from "tabletop-engine";

@State()
class CounterState {
  @field(t.number())
  value = 0;

  increment() {
    this.value += 1;
  }
}

const defineStage = createStageFactory<CounterState>();

// Stage definition (self-referential, so we use a let declaration)
let incrementStage: ReturnType<ReturnType<typeof createStageFactory<CounterState>>>;

incrementStage = defineStage("counting")
  .singleActivePlayer()
  .activePlayer(() => "player1")
  .commands([
    {
      commandId: "increment",
      validate: () => ({ ok: true }),
      execute: ({ game }) => {
        (game as CounterState).increment();
      },
    },
  ])
  .transition(() => incrementStage)
  .build();

const game = new GameDefinitionBuilder("counter")
  .rootState(CounterState)
  .initialStage(incrementStage)
  .build();

const executor = createGameExecutor(game);
const state = executor.createInitialState("seed-123");
const result = executor.executeCommand(state, {
  type: "increment",
  actorId: "player1",
  input: {},
});

if (result.ok) {
  console.log(result.state.game.value); // 1
}
```
