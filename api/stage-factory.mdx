---
title: "Stage factory API reference"
description: "Use createStageFactory to define singleActivePlayer, automatic, and multiActivePlayer stages with chainable, type-safe builder methods."
---

Stages model the distinct phases of your game — whose turn it is, what actions they may take, and where the game goes next. `createStageFactory` returns a factory function you call with a stage ID to get a builder for one of three stage kinds. Each builder exposes chainable methods that accumulate configuration, then a `.build()` call that produces the sealed stage definition.

## `createStageFactory<GameState>()`

```typescript
import { createStageFactory } from "tabletop-engine";

const stage = createStageFactory<MyGameState>();
```

<ParamField path="GameState" type="generic type parameter" required>
  The facade game state type inferred from your root `@State()` class. Pass this so all stage callbacks receive the correct typed game view.
</ParamField>

<ResponseField name="returns" type="StageFactory<GameState>">
  A factory function with the signature `(id: string) => { singleActivePlayer(), automatic(), multiActivePlayer() }`. Call it with a unique stage ID to get a builder for that stage.
</ResponseField>

```typescript
const stage = createStageFactory<MyGameState>();

// Creates a builder for the "player-turn" stage
const builder = stage("player-turn");
```

Each call to the factory returns an object with three builder constructors:

- `builder.singleActivePlayer()` — one player acts at a time
- `builder.automatic()` — the engine runs logic automatically; no player input required
- `builder.multiActivePlayer()` — multiple players submit commands concurrently

## `SingleActivePlayerStageBuilder`

Use this builder when exactly one player may act during a stage.

```typescript
const playerTurn = stage("player-turn")
  .singleActivePlayer()
  .activePlayer(({ game }) => game.currentPlayerId)
  .commands([placeTokenCommand])
  .nextStages(() => ({ nextTurn: playerTurn }))
  .transition(({ nextStages }) => nextStages.nextTurn)
  .build();
```

### `.activePlayer(fn)` <span style={{color:"red"}}>required</span>

Provides the ID of the player who acts during this stage.

<ParamField path="fn" type="(context: SingleActivePlayerSelectionContext) => string" required>
  Called each time the stage machine enters this stage. The context provides:

  - `context.game` — a read-only facade view of the game state
  - `context.runtime` — the current runtime state (read-only)

  Return the player ID string for the actor who should act.
</ParamField>

<ResponseField name="returns" type="SingleActivePlayerStageBuilder (re-typed)">
  The same builder with the `HasActivePlayer` flag set to `true`, unlocking `.build()` once all required methods are called.
</ResponseField>

### `.commands(commands)` <span style={{color:"red"}}>required</span>

Registers the commands that the active player may issue during this stage.

<ParamField path="commands" type="DefinedCommand<GameState>[]" required>
  An array of command definitions produced by `CommandBuilder.build()`. Command IDs must be unique across the entire game; `.build()` on the `GameDefinitionBuilder` enforces this.
</ParamField>

<ResponseField name="returns" type="SingleActivePlayerStageBuilder (re-typed)">
  The builder with `HasCommands` set to `true`.
</ResponseField>

### `.nextStages(resolver)` <span style={{color:"grey"}}>optional</span>

Provides a lazy map of stages the game can transition to from this one. The resolver is called during stage-graph traversal so forward references work without circular import problems.

<ParamField path="resolver" type="() => Record<string, StageDefinition>" required>
  A zero-argument function returning an object whose values are stage definitions. Name the keys whatever is meaningful to your transition logic.
</ParamField>

<ResponseField name="returns" type="SingleActivePlayerStageBuilder (re-typed)">
  The builder with the `NextStages` type parameter updated to the resolved map type.
</ResponseField>

### `.transition(fn)` <span style={{color:"red"}}>required</span>

Determines which stage the game moves to after the active player executes any command in this stage.

<ParamField path="fn" type="(context: SingleActivePlayerTransitionContext) => StageDefinition" required>
  Called after a command executes successfully. The context provides:

  - `context.game` — a read-only facade view of the updated game state
  - `context.runtime` — the updated runtime state (read-only)
  - `context.command` — the typed command that was just executed
  - `context.nextStages` — the resolved next-stage map from `.nextStages()`

  Return any stage definition, including the current stage for a loop.
</ParamField>

<ResponseField name="returns" type="SingleActivePlayerStageBuilder (re-typed)">
  The builder with `HasTransition` set to `true`.
</ResponseField>

### `.build()` <span style={{color:"red"}}>required</span>

Produces the sealed `SingleActivePlayerStageDefinition`.

**Throws** if `.activePlayer()`, `.commands()`, or `.transition()` have not been called.

<ResponseField name="returns" type="SingleActivePlayerStageDefinition">
  Pass this to `.initialStage()` on the `GameDefinitionBuilder`, or reference it from another stage's `.nextStages()` resolver.
</ResponseField>

## `AutomaticStageBuilder`

Use this builder for stages that run engine logic without waiting for player input — for example, dealing cards, resolving end-of-round scoring, or checking win conditions.

```typescript
const dealCards = stage("deal-cards")
  .automatic()
  .run(({ game, rng }) => {
    game.deck = rng.shuffle(game.deck);
    game.hand = game.deck.splice(0, 5);
  })
  .nextStages(() => ({ playerTurn }))
  .transition(({ nextStages }) => nextStages.playerTurn)
  .build();
```

### `.run(fn)` <span style={{color:"grey"}}>optional</span>

Executes side-effecting logic when the stage machine enters this stage.

<ParamField path="fn" type="(context: AutomaticStageRunContext) => void" required>
  Called once when entering the stage. The context provides:

  - `context.game` — a **mutable** facade view of the game state
  - `context.runtime` — the current runtime state (read-only)
  - `context.rng` — the deterministic RNG service
  - `context.emitEvent(event)` — emits a `GameEvent` into the result's event list
</ParamField>

<ResponseField name="returns" type="AutomaticStageBuilder">
  The same builder instance.
</ResponseField>

### `.nextStages(resolver)` <span style={{color:"grey"}}>optional</span>

Provides the stage map for the transition function. Same semantics as on `SingleActivePlayerStageBuilder`.

<ParamField path="resolver" type="() => Record<string, StageDefinition>" required>
  A zero-argument resolver returning named stage definitions.
</ParamField>

<ResponseField name="returns" type="AutomaticStageBuilder (re-typed)">
  The builder with the `NextStages` type updated.
</ResponseField>

### `.transition(fn)` <span style={{color:"grey"}}>optional</span>

Determines where the game goes after `.run()` completes. If omitted, the stage machine stops at this stage.

<ParamField path="fn" type="(context: AutomaticStageTransitionContext) => StageDefinition" required>
  The context provides:

  - `context.game` — a read-only facade view
  - `context.runtime` — the updated runtime state (read-only)
  - `context.nextStages` — the resolved next-stage map
</ParamField>

<ResponseField name="returns" type="AutomaticStageBuilder">
  The same builder instance.
</ResponseField>

### `.build()`

Produces the sealed `AutomaticStageDefinition`. Unlike the other builders, no methods are required before calling `.build()` — an automatic stage with no `run` and no `transition` is valid as a terminal stage.

<ResponseField name="returns" type="AutomaticStageDefinition">
  The sealed stage definition.
</ResponseField>

## `MultiActivePlayerStageBuilder`

Use this builder when multiple players must each submit a command before the game advances — for example, a simultaneous bidding round or a draft pick.

```typescript
const biddingRound = stage("bidding-round")
  .multiActivePlayer()
  .memory(
    t.object({ bids: t.record(t.string(), t.number()) }),
    () => ({ bids: {} }),
  )
  .activePlayers(({ game }) => game.playerIds)
  .commands([placeBidCommand])
  .onSubmit(({ memory, command }) => {
    memory.bids[command.actorId] = command.input.amount;
  })
  .isComplete(({ game, memory }) =>
    game.playerIds.every((id) => id in memory.bids),
  )
  .nextStages(() => ({ resolve: resolveRound }))
  .transition(({ nextStages }) => nextStages.resolve)
  .build();
```

### `.memory(schema, factory)` <span style={{color:"red"}}>required</span>

Declares the shared memory object for this stage. Memory persists across all player submissions within a single round of this stage and is serialized as part of the runtime state.

<ParamField path="schema" type="ObjectFieldType<Record<string, FieldType>>" required>
  A serializable object schema built with `t.object(...)`. The schema must be serializable (no function types, no class instances).
</ParamField>

<ParamField path="factory" type="() => Memory" required>
  A zero-argument function that returns the initial memory value each time this stage is entered.
</ParamField>

<ResponseField name="returns" type="MultiActivePlayerStageBuilder (re-typed)">
  The builder with `HasMemory` set to `true` and `Memory` typed to the factory's return type.
</ResponseField>

### `.activePlayers(fn)` <span style={{color:"red"}}>required</span>

Provides the list of player IDs who must each submit a command before the stage can advance.

<ParamField path="fn" type="(context: MultiActivePlayerMemoryContext) => string[]" required>
  Called when entering the stage and after each submission. The context provides:

  - `context.game` — a read-only facade view
  - `context.runtime` — the current runtime state (read-only)
  - `context.memory` — the current stage memory

  Return the array of player IDs who are still expected to act.
</ParamField>

<ResponseField name="returns" type="MultiActivePlayerStageBuilder (re-typed)">
  The builder with `HasActivePlayers` set to `true`.
</ResponseField>

### `.commands(commands)` <span style={{color:"red"}}>required</span>

Registers the commands available to active players during this stage.

<ParamField path="commands" type="DefinedCommand<GameState>[]" required>
  An array of command definitions produced by `CommandBuilder.build()`.
</ParamField>

<ResponseField name="returns" type="MultiActivePlayerStageBuilder (re-typed)">
  The builder with `HasCommands` set to `true`.
</ResponseField>

### `.onSubmit(fn)` <span style={{color:"red"}}>required</span>

Called each time one of the active players submits a command. Use this to record the submission in stage memory and optionally execute game state mutations.

<ParamField path="fn" type="(context: MultiActivePlayerSubmitContext) => void" required>
  The context provides:

  - `context.game` — a read-only facade view
  - `context.runtime` — the current runtime state (read-only)
  - `context.memory` — the shared stage memory (mutable)
  - `context.command` — the typed command submitted by the player
  - `context.execute(command)` — call this to run a command's `execute` function against the game state immediately
</ParamField>

<ResponseField name="returns" type="MultiActivePlayerStageBuilder (re-typed)">
  The builder with `HasOnSubmit` set to `true`.
</ResponseField>

### `.isComplete(fn)` <span style={{color:"red"}}>required</span>

Determines whether all expected submissions have been received, triggering the transition to the next stage.

<ParamField path="fn" type="(context: MultiActivePlayerMemoryContext) => boolean" required>
  Called after each submission. Return `true` to advance, `false` to wait for more submissions.

  The context provides `game`, `runtime`, and `memory`.
</ParamField>

<ResponseField name="returns" type="MultiActivePlayerStageBuilder (re-typed)">
  The builder with `HasIsComplete` set to `true`.
</ResponseField>

### `.nextStages(resolver)` <span style={{color:"red"}}>required</span>

Provides the stage map for the transition function. Required for `multiActivePlayer` (unlike the other builders where it is optional).

<ParamField path="resolver" type="() => Record<string, StageDefinition>" required>
  A zero-argument resolver returning named stage definitions.
</ParamField>

<ResponseField name="returns" type="MultiActivePlayerStageBuilder (re-typed)">
  The builder with `HasNextStages` set to `true`.
</ResponseField>

### `.transition(fn)` <span style={{color:"red"}}>required</span>

Determines where the game goes after `isComplete` returns `true`.

<ParamField path="fn" type="(context: MultiActivePlayerTransitionContext) => StageDefinition" required>
  The context provides:

  - `context.game` — a read-only facade view of the game state after all submissions
  - `context.runtime` — the updated runtime state (read-only)
  - `context.memory` — the final stage memory
  - `context.nextStages` — the resolved next-stage map
</ParamField>

<ResponseField name="returns" type="MultiActivePlayerStageBuilder (re-typed)">
  The builder with `HasTransition` set to `true`.
</ResponseField>

### `.build()` <span style={{color:"red"}}>required</span>

Produces the sealed `MultiActivePlayerStageDefinition`.

**Throws** if any of `.memory()`, `.activePlayers()`, `.commands()`, `.onSubmit()`, `.isComplete()`, `.nextStages()`, or `.transition()` have not been called.

<ResponseField name="returns" type="MultiActivePlayerStageDefinition">
  The sealed stage definition.
</ResponseField>
