---
title: "Testing Harness: runScenario and ScenarioResult"
description: "runScenario() executes an ordered list of commands against a GameExecutor and returns ScenarioResult with initial state, final state, and each ExecutionResult."
---

The testing harness gives you a single function, `runScenario()`, that wires together initial state creation, sequential command execution, and result collection. Instead of manually threading state through a loop, you hand `runScenario` an executor, a list of commands, and the arguments for `createInitialState`, and it returns a structured `ScenarioResult` with the initial state, the final state, and every intermediate `ExecutionResult`. This keeps test bodies short and declarative.

## `runScenario(executor, commands, ...initialStateArgs)`

```typescript
function runScenario<
  State extends CanonicalState,
  TCommandInput extends Command,
>(
  gameExecutor: {
    createInitialState(...args: unknown[]): State;
    executeCommand(state: State, command: TCommandInput): ExecutionResult<State>;
  },
  commands: TCommandInput[],
  ...initialStateArgs: unknown[]
): ScenarioResult<State, TCommandInput>
```

Creates the initial state, executes each command in `commands` in order, and returns a `ScenarioResult` containing the initial state, the final state, all commands, and each corresponding `ExecutionResult`.

<ParamField path="executor" type="object" required>
  Any object with `createInitialState` and `executeCommand` methods. A
  `GameExecutor` returned by `createGameExecutor()` satisfies this
  interface.
</ParamField>

<ParamField path="commands" type="TCommandInput[]" required>
  An ordered list of commands to execute. They are applied one at a time;
  each command receives the state produced by the previous command.
</ParamField>

<ParamField path="...initialStateArgs" type="unknown[]">
  Forwarded directly to `executor.createInitialState()`. Pass any arguments
  your game's setup function expects — player IDs, seeds, rule variants, etc.
</ParamField>

## `ScenarioResult<State, Command>`

```typescript
interface ScenarioResult<
  State extends CanonicalState = CanonicalState,
  TCommandInput extends Command = Command,
> {
  initialState: State;
  finalState: State;
  commands: TCommandInput[];
  results: ExecutionResult<State>[];
}
```

<ResponseField name="initialState" type="State">
  The state returned by `createInitialState`, before any commands have been
  applied.
</ResponseField>

<ResponseField name="finalState" type="State">
  The state after all commands in `commands` have been applied in order.
  If `commands` is empty, `finalState === initialState`.
</ResponseField>

<ResponseField name="commands" type="TCommandInput[]">
  The original `commands` array passed to `runScenario`, unchanged.
</ResponseField>

<ResponseField name="results" type="ExecutionResult<State>[]">
  An `ExecutionResult` for each command in order. `results[i]` corresponds to
  `commands[i]`. Each result is either an `ExecutionSuccess` (`ok: true`,
  carries `state` and `events`) or an `ExecutionFailure` (`ok: false`, carries
  `state`, `reason`, and `events`).
</ResponseField>

## Example

```typescript
import { test, expect } from "bun:test";
import { runScenario, createGameExecutor } from "tabletop-engine";
import { createMyGame } from "./game";

const executor = createGameExecutor(createMyGame());

test("player can increment counter", () => {
  const scenario = runScenario(
    executor,
    [{ type: "increment", actorId: "p1", input: {} }],
    "seed-123",
  );

  expect(scenario.results[0].ok).toBe(true);
  expect(scenario.finalState.game.value).toBe(1);
});

test("two players can take turns", () => {
  const scenario = runScenario(
    executor,
    [
      { type: "take_gems", actorId: "p1", input: { colors: ["white", "blue"] } },
      { type: "reserve_card", actorId: "p2", input: { cardId: 7 } },
    ],
    { playerIds: ["p1", "p2"] },
    "seed-abc",
  );

  expect(scenario.results[0].ok).toBe(true);
  expect(scenario.results[1].ok).toBe(true);
  expect(scenario.finalState.game.players.p2?.reservedCardIds).toContain(7);
});
```

## Checking failures

`runScenario` applies every command regardless of whether earlier commands succeed. If a command returns `ok: false`, the state passed to the next command is the unchanged state from that failed result. Check `results[i].ok` to distinguish success from failure:

```typescript
test("second command fails if first command exhausts the resource", () => {
  const scenario = runScenario(
    executor,
    [
      { type: "take_all_tokens", actorId: "p1", input: {} },
      { type: "take_all_tokens", actorId: "p2", input: {} },
    ],
    { playerIds: ["p1", "p2"] },
    "seed-123",
  );

  expect(scenario.results[0].ok).toBe(true);
  expect(scenario.results[1].ok).toBe(false);

  if (!scenario.results[1].ok) {
    expect(scenario.results[1].reason).toBe("no_tokens_remaining");
  }
});
```

## Accessing initial state in assertions

`scenario.initialState` is the state before any commands run. Use it to assert that commands produce the expected delta:

```typescript
test("score increases by the correct amount", () => {
  const scenario = runScenario(
    executor,
    [{ type: "buy_card", actorId: "p1", input: { cardId: 42 } }],
    { playerIds: ["p1", "p2"] },
    "seed-xyz",
  );

  const before = scenario.initialState.game.players.p1?.score ?? 0;
  const after = scenario.finalState.game.players.p1?.score ?? 0;

  expect(after - before).toBe(3);
});
```

<Tip>
  For long multi-command sequences, destructure `results` directly:
  `const [firstResult, secondResult] = scenario.results;`. This avoids
  repeated indexing and keeps assertion lines readable.
</Tip>
