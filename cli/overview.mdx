`tabletop-cli` is the companion CLI for `tabletop-engine`. Point it at your game definition and it produces everything your server and clients need: JSON schemas for canonical and visible state, a protocol descriptor, TypeScript type declarations, and a fully typed WebSocket client SDK. You run these commands once during development and again whenever your game's commands or state shape changes.

## Installation

<CodeGroup>

```bash bun
bun add tabletop-cli
```

```bash npm
npm install tabletop-cli
```

</CodeGroup>

## Configuration

All commands read your game definition from a `tabletop.config.ts` file in the current working directory. Export your game using `defineConfig` from `tabletop-engine/config`:

```typescript tabletop.config.ts
import { defineConfig } from "tabletop-engine/config";
import { createMyGame } from "./src/game.ts";

export default defineConfig({
  game: createMyGame(),
  outDir: "./generated",
});
```

The `game` field is required and must be a built `GameDefinition`. The `outDir` field is optional — when omitted, the CLI writes output to a `generated/` directory next to your config file.

## Available commands

| Command | Description |
|---------|-------------|
| [`generate`](/cli/generate) | Generate schemas, types, protocol, or a client SDK |
| [`validate`](/cli/validate) | Validate your game definition and optional state snapshots |

## Global flags

Both commands accept the following flags:

<ParamField path="--config" type="string">
  Path to a config file. Defaults to `tabletop.config.ts` in the current working directory.

  ```bash
  tabletop-cli generate schemas --config ./config/tabletop.config.ts
  ```
</ParamField>

<ParamField path="--outDir" type="string">
  Override the output directory for generated files. Relative paths are resolved from the current working directory. Takes precedence over `outDir` in your config file.

  ```bash
  tabletop-cli generate types --outDir ./src/generated
  ```
</ParamField>

<Note>
  The `--outDir` flag is available on `generate` subcommands. The `validate` command uses `--snapshot` instead — see [validate](/cli/validate) for details.
</Note>
