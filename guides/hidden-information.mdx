title: "Implementing hidden information in your game"
description: "Use configureVisibility() to project player-specific views of state, hiding opponent hands and secret decks while exposing sanitized metadata."
---

Most tabletop games require that some information stay private — a player's hand of cards, the cards they've reserved, or the contents of a face-down deck. tabletop-engine separates your authoritative canonical state (which the server holds) from the visible state each viewer receives. You define the projection rules once with `configureVisibility()`, and the executor handles the rest when you call `getView()`.

## Canonical state vs. visible state

The canonical state is the single source of truth maintained by the executor. It contains everything — every player's hand, every deck's order, every hidden score. No player ever receives the canonical state directly.

When you call `executor.getView(state, viewer)`, the engine walks the compiled state facade and applies your visibility rules to produce a viewer-specific snapshot. Fields you've marked as hidden are replaced with an envelope object:

```typescript
{ __hidden: true, value?: <derived-metadata> }
```

Fields you haven't configured at all remain fully visible to all viewers.

## Configuring visibility

Call `configureVisibility()` after defining your state class. It takes the class itself and a builder callback that receives a typed `field` proxy:

```typescript
import { configureVisibility, field, State, t } from "tabletop-engine";

@State()
export class PlayerState {
  @field(t.string())
  id = "";

  @field(t.array(t.number()))
  handCardIds: number[] = [];

  @field(t.number())
  score = 0;
}

configureVisibility(PlayerState, ({ field }) => ({
  ownedBy: field.id,
  fields: [
    field.handCardIds.hidden(),
    field.score.visibleToSelf(),
  ],
}));
```

### `ownedBy`

Set `ownedBy` to a string field that holds the player ID this state instance belongs to. The engine uses it to determine which player is the "self" when evaluating `visibleToSelf()` rules.

### `hidden()`

A field marked `.hidden()` is replaced with `{ __hidden: true }` for every viewer — including the owning player. Use this for information that no one should see at runtime, such as the order of a face-down deck.

### `visibleToSelf()`

A field marked `.visibleToSelf()` is visible only to the player identified by `ownedBy`. All other viewers see `{ __hidden: true }`. Use this for private hands, reserved cards, or per-player secrets.

## Deriving sanitized metadata

Both `.hidden()` and `.visibleToSelf()` accept an optional `{ schema, derive }` pair. Instead of the raw `{ __hidden: true }` envelope, viewers receive the derived value embedded as `{ __hidden: true, value: <derived> }`. This lets you share useful metadata — like a count — without leaking the underlying data.

```typescript
const hiddenHandSchema = t.object({ count: t.number() });

configureVisibility(PlayerState, ({ field }) => ({
  ownedBy: field.id,
  fields: [
    field.handCardIds.visibleToSelf({
      schema: hiddenHandSchema,
      derive(handCardIds) {
        return { count: handCardIds.length };
      },
    }),
  ],
}));
```

Opponents now see `{ __hidden: true, value: { count: 3 } }` instead of the raw card IDs. The owning player still sees the full array.

<Note>
When you provide `schema`, you must also provide `derive`. The engine enforces this at configuration time and will throw `visibility_schema_requires_derive` if `derive` is absent.
</Note>

## A card hand example

Here's the full pattern from the Splendor reference implementation. Reserved cards are private to each player; opponents can see how many cards a player has reserved without seeing which cards they are.

```typescript
import { configureVisibility, field, State, t } from "tabletop-engine";

const hiddenReservedCardSchema = t.object({
  count: t.number(),
});

@State()
export class SplendorPlayerState {
  @field(t.string())
  id = "";

  @field(t.array(t.number()))
  reservedCardIds: number[] = [];

  @field(t.array(t.number()))
  purchasedCardIds: number[] = [];

  // ... other fields
}

configureVisibility(SplendorPlayerState, ({ field }) => ({
  ownedBy: field.id,
  fields: [
    field.reservedCardIds.visibleToSelf({
      schema: hiddenReservedCardSchema,
      derive(reservedCardIds) {
        return { count: reservedCardIds.length };
      },
    }),
  ],
}));
```

The board's deck order is completely hidden from everyone — only the count per level is exposed:

```typescript
const hiddenDeckSchema = t.object({
  1: t.number(),
  2: t.number(),
  3: t.number(),
});

configureVisibility(SplendorBoardState, ({ field }) => ({
  fields: [
    field.deckByLevel.hidden({
      schema: hiddenDeckSchema,
      derive(deckByLevel) {
        return {
          1: deckByLevel[1]?.length ?? 0,
          2: deckByLevel[2]?.length ?? 0,
          3: deckByLevel[3]?.length ?? 0,
        };
      },
    }),
  ],
}));
```

## Projecting state with getView()

Pass a `Viewer` object to `executor.getView()` to produce the viewer-specific snapshot:

```typescript
// Player viewer
const aliceView = executor.getView(state, {
  kind: "player",
  playerId: "alice",
});

// Spectator viewer — sees the same as a player with no ownership
const spectatorView = executor.getView(state, {
  kind: "spectator",
});
```

<CodeGroup>

```typescript Player view
// Alice sees her own reserved cards as a full array
aliceView.game.players.alice.reservedCardIds;
// => [12, 37, 58]

// Alice sees Bob's reserved cards as a hidden envelope with metadata
aliceView.game.players.bob.reservedCardIds;
// => { __hidden: true, value: { count: 2 } }

// The deck order is hidden from everyone
aliceView.game.board.deckByLevel;
// => { __hidden: true, value: { 1: 12, 2: 5, 3: 3 } }
```

```typescript Spectator view
// Spectators see all reserved cards as hidden
spectatorView.game.players.alice.reservedCardIds;
// => { __hidden: true, value: { count: 3 } }

spectatorView.game.players.bob.reservedCardIds;
// => { __hidden: true, value: { count: 2 } }
```

</CodeGroup>

<Tip>
`getView()` does not mutate state — it produces a new plain object. You can call it as often as needed, for example once per connected client on each state change.
</Tip>

## Viewer types

| Viewer | Type literal | Behavior |
| --- | --- | --- |
| A specific player | `{ kind: "player", playerId: "alice" }` | `visibleToSelf()` fields are visible for states owned by `alice` only |
| Spectator | `{ kind: "spectator" }` | Treated as having no ownership; all `visibleToSelf()` fields are hidden |
